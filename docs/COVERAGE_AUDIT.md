# API Coverage Audit — 냉정한 진단

> 2026-04-07 기준. "364 tools"의 실체를 파헤친다.

## 핵심 결론

**364개 도구라고 하지만, 주요 데이터 소스의 실제 커버리지는 1~30%.**
대부분의 API 엔드포인트는 노출되지 않거나, 제네릭 도구 뒤에 숨어있어 사용자가 존재를 모른다.

## 소스별 상세 진단

### 1. DART — 가장 심각 (17% 커버리지)

**있는 것 (7개)**:
- 기업개황, 재무제표, 재무비율, 대주주, 현금흐름, 배당, 기업검색

**없는 것 (33+ 엔드포인트)**:

| 카테고리 | 빠진 데이터 | 중요도 | 구현 난이도 |
|----------|-------------|--------|-------------|
| **공시 검색** | 공시목록 검색/필터 | 최상 | 쉬움 |
| **임원** | 임원현황, 임원보수 | 상 | 쉬움 (1줄) |
| **지배구조** | 최대주주변동, 공정공시(특수관계) | 상 | 쉬움 (1줄) |
| **기업활동** | 합병분할, 증자감자 | 상 | 쉬움 (1줄) |
| **자본** | 전환사채, 자기주식 | 중 | 쉬움 (1줄) |
| **위험** | 부도/회생, 지분공시(5%룰) | 중 | 쉬움 (1줄) |

**치명적 문제**: `_dart.report(code, section, year)` 메서드가 10개 섹션을 지원하는데,
우리는 **"배당" 1개만** 사용 중. 나머지 9개는 코드 1줄이면 추가 가능.

### 2. CCXT — 두 번째로 심각 (1% 커버리지)

**있는 것 (8개)**:
- 시세, 오더북, OHLCV, 김프, 거래소비교, 거래량, 스프레드

**없는 것**:
| 빠진 기능 | 영향 |
|-----------|------|
| 심볼/거래쌍 검색 | "바이낸스에서 뭘 거래할 수 있어?" 대답 불가 |
| 거래소 목록 | 100+ 거래소 중 7개만 하드코딩 |
| 체결 내역 (trades) | 최근 거래 데이터 접근 불가 |
| 펀딩 레이트 (perp) | crypto_quant에 일부 있지만 직접 접근 불가 |
| 인증된 엔드포인트 | 잔고/주문/입출금 전부 불가 |
| 레버리지/포지션 | 선물 거래 데이터 접근 불가 |

### 3. ECOS — 부분 개방 (30% 커버리지)

**ECOS_STAT_CODES에 30개 지표 하드코딩**:
- 금리(8), 통화(2), 환율(1), GDP(1), 물가(2), 고용(3), 가계(2), 국제수지(2), 경기(3), 부동산(2), 자본시장(2), 산업(1)

**빠진 것 (70+ 통계표)**:
- 금융조건지수, 선행지수, 부가가치세, 가처분소득
- 민간투자, 정부지출, 채권발행, 기업부도율
- 반도체/철강/자동차 산업별 수출
- 신용카드 사용, 전력소비 등

**구원 도구**: `ecos_search_stat_list` + `ecos_get_stat_data`로 임의 통계 조회 가능.
하지만 사용자가 이 도구의 존재와 사용법을 모름.

### 4. SEC — 제한적 (5-10%)

**XBRL 추출이 10개 필드로 제한**:
- Revenues, NetIncome, Assets, Liabilities, EPS, Cash, CashFlow 등
- SEC XBRL은 1000+ fact type 보유
- 임의 concept 쿼리 불가

### 5. Global Macro — 코드 지식 필요 (1%)

**제네릭 도구 있지만**:
- `macro_oecd(dataset="MEI", ...)` 처럼 데이터셋 코드를 알아야 함
- 지표 검색 도구 없음
- OECD API 2024년 변경으로 일부 깨짐

## 구조적 문제 5가지

### 1. "하드코딩 함정"
대부분의 서버가 특정 지표만 하드코딩. 새 데이터 필요 → 코드 수정 필요.
**해결**: 제네릭 쿼리 도구 + 검색/탐색 도구 조합

### 2. "숨겨진 제네릭 도구"
`ecos_get_stat_data`, `kosis_get_table`, `energy_eia_series` 등 강력한 도구가 있지만
사용자가 존재를 모름. docstring에만 설명, 사용 예시 없음.
**해결**: 문서화 + 도구 설명 보강

### 3. "검색 없는 API"
CCXT, DART, Global Macro에는 검색/탐색 도구가 아예 없음.
사용자가 "뭐가 있는지" 알 방법이 없음.
**해결**: 각 서버에 discovery 도구 추가

### 4. "report() 1/10 사용"
DART의 report() 메서드는 10개 섹션(배당/임원/M&A/증자/합병 등)을 지원하지만
1개(배당)만 사용. 나머지 9개는 **코드 1줄**이면 추가 가능.
**해결**: 즉시 추가 (effort: 최소)

### 5. "인증 엔드포인트 0%"
CCXT(거래), SEC(프리미엄), EIA(엔터프라이즈) 등
인증이 필요한 엔드포인트는 전혀 노출되지 않음.

## 즉시 실행 가능한 개선 (코드 변경 최소)

### Tier 1: 1줄 추가로 가능 (DART report 확장)

| 새 도구 | 코드 | 예상 줄수 |
|---------|------|-----------|
| `dart_executives` | `report(code, "임원현황", year)` | ~15줄 |
| `dart_executive_compensation` | `report(code, "임원보수", year)` | ~15줄 |
| `dart_shareholder_changes` | `report(code, "최대주주변동", year)` | ~15줄 |
| `dart_corporate_actions` | `report(code, "합병분할", year)` + `report(code, "증자감자", year)` | ~25줄 |
| `dart_related_party` | `report(code, "공정공시", year)` | ~15줄 |
| `dart_treasury_stock` | `report(code, "자기주식", year)` | ~15줄 |
| `dart_convertible_bonds` | `report(code, "전환사채", year)` | ~15줄 |
| `dart_insolvency` | `report(code, "부도회생", year)` | ~15줄 |
| `dart_5pct_disclosure` | `report(code, "지분공시", year)` | ~15줄 |
| **합계** | | **~145줄 추가** |

→ DART 커버리지: **17% → 60%+** (9개 도구 추가)

### Tier 2: 검색/탐색 도구 추가 (각 서버)

| 서버 | 새 도구 | 효과 |
|------|---------|------|
| CCXT | `crypto_list_symbols(exchange)` | 거래쌍 탐색 가능 |
| CCXT | `crypto_list_exchanges()` | 100+ 거래소 노출 |
| CCXT | `crypto_recent_trades(exchange, symbol)` | 체결 내역 |
| DART | `dart_disclosure_search(keyword, date_from)` | 공시 검색 |
| SEC | `sec_xbrl_concept(ticker, concept)` | 임의 XBRL 팩트 |
| Global Macro | `macro_search_indicators(source, keyword)` | 지표 검색 |

### Tier 3: ECOS 통계코드 확장

`ECOS_STAT_CODES`에 70+ 통계표 추가:
- 금융조건지수, 선행지수, 가처분소득, 민간투자
- 산업별 수출(반도체/자동차/철강), 전력소비
- 이미 정의된 30개 + 추가 70개 = 100개 커버
- 순수 dict 항목 추가 (코드 변경 없음)
